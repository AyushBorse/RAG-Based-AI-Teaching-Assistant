# 🎓 RAG-Based AI Teaching Assistant

A fully local, offline-capable Retrieval-Augmented Generation (RAG) pipeline that turns Python course videos into a conversational teaching assistant. Ask any question about the course and get a precise answer — complete with the exact video and timestamp where the topic is explained.

---

## 🧠 How It Works

The system is a 4-stage pipeline:

```
Videos (.mp4/.mkv)
     │
     ▼
[1] video_to_mp3.py       →  Extract audio using FFmpeg
     │
     ▼
[2] mp3_to_json.py        →  Transcribe & translate with Whisper (large-v2)
     │                        (Hindi → English, timestamped segments)
     ▼
[3] preprocess_json.py    →  Generate embeddings via Ollama (bge-m3)
     │                        Save vector store as embeddings.joblib
     ▼
[4] process_incoming.py   →  Accept user query → embed → cosine similarity
                              → retrieve top-5 chunks → LLM (Llama 3.2)
                              → respond with video + timestamp references
```

## ⚙️ Tech Stack

| Component | Tool |
|---|---|
| Audio Extraction | `FFmpeg` |
| Speech-to-Text | `OpenAI Whisper` (large-v2) |
| Embedding Model | `Ollama` + `bge-m3` |
| LLM for Q&A | `Ollama` + `Llama 3.2` |
| Vector Search | `scikit-learn` (cosine similarity) |
| Data Handling | `pandas`, `numpy`, `joblib` |

> **All models run locally via Ollama — no API keys or internet required after setup.**

## 🚀 Setup & Usage

### Prerequisites

- Python 3.9+
- [FFmpeg](https://ffmpeg.org/download.html) installed and in PATH
- [Ollama](https://ollama.com/) installed and running

### 1. Install Python Dependencies

```bash
pip install openai-whisper requests numpy pandas scikit-learn joblib
```

### 2. Pull Required Ollama Models

```bash
ollama pull bge-m3
ollama pull llama3.2
```

### 3. Run the Pipeline

**Stage 1 — Convert videos to audio:**
```bash
# Place your videos in the `videos/` folder first
python video_to_mp3.py
```
> Expects filenames in the format: `Python Tutorial #5 [abc] | Lists | ...`

**Stage 2 — Transcribe & translate audio:**
```bash
python mp3_to_json.py
```
> Transcribes Hindi audio and translates to English using Whisper `large-v2`. Saves timestamped chunk JSONs to `jsons/`.

**Stage 3 — Generate embeddings:**
```bash
python preprocess_json.py
```
> Reads all JSONs, generates `bge-m3` embeddings per chunk via Ollama, and saves the vector store as `embeddings.joblib`.

**Stage 4 — Ask questions:**
```bash
python process_incoming.py
```
```
Ask a Question: How do I use list comprehension in Python?
```
The assistant will return an answer like:
> *"List comprehension is covered in Tutorial #7 - Lists, starting around 4:30. The instructor demonstrates how to write `[x*2 for x in range(10)]` as a concise alternative to a for loop..."*

## 📦 Data Format

Each transcription chunk stored in the JSON files has this structure:

```json
{
  "number": "7",
  "title": "Lists",
  "start": 270.5,
  "end": 285.0,
  "text": "So here we use list comprehension to double every element..."
}
```

The vector store (`embeddings.joblib`) is a pandas DataFrame with an additional `embedding` column (1024-dim `bge-m3` vector) and a `chunk_id`.

---

## 🔍 RAG Query Flow

When a question is asked in `process_incoming.py`:

1. The question is embedded using `bge-m3` via Ollama.
2. Cosine similarity is computed against all stored chunk embeddings.
3. The **top 5 most relevant chunks** are retrieved.
4. A structured prompt is built with chunk metadata (title, number, start, end, text).
5. The prompt is sent to **Llama 3.2** running locally via Ollama.
6. The model responds in natural language, pointing the user to the relevant video and timestamp.

---

## 🌐 Notes

- Video filenames must follow the naming convention `<name> #<number> [id] | <title> | ...` for correct parsing in `video_to_mp3.py` and `mp3_to_json.py`.
- The Whisper model is set to `language="hi"` (Hindi) with `task="translate"` (to English). Change these in `mp3_to_json.py` for other languages.
- Make sure `ollama serve` is running before executing Stage 3 or Stage 4.
- Stages 1–3 are one-time preprocessing steps. Only Stage 4 needs to be re-run for each query.
